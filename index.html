内存管理笔记

发表于 2018-01-18 | 分类于 笔记

OC中的内存管理

oc作为面向对象的c,其内存也是自己管理的,但oc中对其操作进行了优化
从c的自己开辟和回收内存,改进为手动计算引用数

当引用增加时 调用retain方法为此变量引用数+1
当要释放引用时 调用release方法为引用数-1
当引用数为0时将释放内存

在xcode4后 引入了ARC模式 xcode将在编译自动时为各个对象引用数量改变时插入 retain和release方法
从此解放双手
对循环引用的解决方案为在编写代码时 一方使用强引用 一方使用弱引用

python的垃圾回收

  • 自动引用计数

    对象每增加一个引用 引用计数+1
    减少引用计数-1 当引用计数为0删除对象

    缺点:无法清理循环引用

  • 隔代回收

    python每个对象创建时 ,都会将其加入零代链表 在满足一定条件(创建对象计数-回收对象大于一定数量)后
    判断链表中对象是否有互相引用 有的引用计数-1 引用计数为0的删除 引用计数大于1且无互相引用的 加入到 一代链表

Java的垃圾回收

  • 可达性分析

    以 GC Root对象为起点向下遍历所有对象为可用对象,其余为不可用对象

    GC Root对象包括下面几种:

    1. 栈中局部变量表中引用的对象
    2. 方法区中静态属性引用的对象
    3. 方法区常量引用的对象
    4. 本地方法栈中JNI引用对象

  • 标记-清除算法

    分为标记,清除两个阶段:首先标记出所有需要回收的对象,标记完成后统一回收所有被标记的对象

    缺点:标记和清除效率都低,且产生大量不连续内存

  • 复制算法( 新生代回收算法 )

    将内存分为2小1大3块为8:1:1 ,每次只用其中一大一小,当这触发回收时,将其中存活的对象复制到另一块未使用的小内存去,然后将已经使用过的内存空间一次性清理掉

    当一些对象经过多次Minor Gc后还仍然存活就会被移动到老年代中

  • 标记-整理算法 ( 老年代回收算法 )

    让存活对象统一向一端移动(不可用对象所占内存在移动时被覆盖),然后直接清除边界以外的内存空间
